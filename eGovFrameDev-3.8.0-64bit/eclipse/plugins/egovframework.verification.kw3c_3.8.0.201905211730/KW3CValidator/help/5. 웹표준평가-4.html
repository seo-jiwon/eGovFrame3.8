웹 표준 평가

평가 결과 보기  

 

평가가 완료되면 자동적으로 평가 결과 보기 창으로 전환됩니다. 프로그램에서 [열기]아이콘을 눌러 프로젝트를 선택하면 해당 프로젝트에 관한 평가 결과를 보여주는 창으로 전환됩니다. 프로젝트를 선택하여 평가 결과를 보는 방법은 기존 평가 결과 보기에서 설명합니다.  

 

 

 

점검결과 창은 3개의 영역으로 구성되어 있습니다.  

1) URL 목록(①) : 웹표준 평가시에 수집된 URL 목록을 보여주는 영역입니다. 만약 CSS 평가를 함께 수행하였다면 CSS 평가 대상 URL을 웹문서(HTML URL)와 함께 표시하며, 이 때 CSS 문서는 웹문서의 하위 노드로 표시됩니다. 여기서 URL 혹은 CSS를 클릭하면 해당 소스를 영역 ②에 보여줍니다.

URL 목록은 상하 이동 키를 이용하여 키보드로도 선택이 가능하며 Space 키를 누르면 해당 URL(또는 CSS)을 마우스로 클릭한 것과 같이 해당 URL에 대한 소스와 평가 결과를 각 영역에 표시됩니다.

URL 목록의 웹 페이지 URL의 [20/30, 15/20] 과 같이 대괄호로 표시된 값은 해당 URL에 대한 DTD 평가 결과 및 CSS 평가 결과입니다. DTD 및 CSS 평가 점수 각각 30점 및 20점 만점 기준으로 표시합니다.

CSS 목록의 (3/2/4)와 같이 소괄호로 표시된 값은 해당 CSS 페이지의 평가 결과 심각, 위함 및 보통에 해당하는 오류의 수를 순서대로 표시한 것입니다. 따라서 (3/2/4)는 심각, 위험 및 보통에 해당하는 오류가 각각 3개, 2개 및 4개임을 의미합니다.

※ 참고: 상태 바에 표시된 정보는 DTD 평가 및 CSS 평가 결과 해당 URL의 평가 결과를 토대로 오류 수를 심각, 위험 및 보통으로 구분하여 표시한 것입니다. 해당 프로젝트의 최종 평가 점수는 URL 목록(①)의 첫 번째 노드(프로젝트 이름)에 표시됩니다.

2) 소스 보기(②) : 우측의 URL 목록 영역(①)에서 선택된 URL과 하위 CSS를 각각 페이지 탭으로 구분하여 표시합니다. URL목록에서 선택한 항목의 페이지 탭이 전면에 나타냅니다. 필요시 수동으로 페이지를 변경할 수 있습니다. 또한 오류목록(③)에서 선택한 항목에 해당하는 소스 페이지로 자동 전환되고 하당 항목을 마우스로 클릭하고 있는 동안 또는 Space 키를 누르고 있는 동안 오류가 발생한 소스 부분을 하이라이트하여 보여줍니다.   자세한 사항은 접근성 기능에서 참고하시기 바랍니다.

3) 오류 결과(③) : 우측의 URL 목록 영역(①)에서 선택된 URL과 하위 CSS에 대한 평가 결과의 오류 내용을 표로 보여줍니다. 각 표의 내용은 다음과 같습니다.

  • 유형 : 오류의 종류를 이미지로 구분하여 보여줍니다. 정보제공, 경고, 오류로 구분합니다.
  • 파일명 : 해당 소스가 저장되어있는 파일명으로 참고용입니다.
  • 라인 : 소스에서 오류가 발견된 해당 라인 번호
  • 열 : 소스에서 오류가 발견된 라인의 해당 열 번호
  • 룰번호 : W3C에서 제시한 오류/경고 문구에 해당하는 번호
  • 점수 : 웹표준 평가 결과 해당 오류로 인한 점수입니다. 심각(-20점), 위험(-3점), 보통(-1점) 등으로 구분됩니다.
  • 오류내용 : 오류의 자세한 내용이 기록되어 있습니다.  

※ 참고로 HTML5의 평가 결과에는 룰번호, 점수 등이 표시되지 않습니다. 그 이유는 HTML5에 대한 표준이 완성되지 않아 룰이 확정되지 않아 점수 산출이 불가능하기 떄문입니다.  

※ 오류 영역이 선택된 상태에서는 상하좌우 키를 이용하여 표의 셀(cell)간 이동이 가능합니다. 특정한 셀이 선택된 상태에서 Space 키를 누르고 있으면 해당 항목을 소스 영역에 하이라이트해 줍니다. Space키를 놓으면 하이라이트가 사라집니다. 마찬가지로 마우스 좌특 버튼을 누르고 있으면 해당 소스를 하이라이트 해주나 마우스 왼쪽 버튼을 누르면 하이라이트가 사라집니다. 이 기능을 이용하면 키보드 사용자와 마우스 사용자가 모두 이프로그램을 사용할 수 있습니다.
